# 🤖 ROS2 + FastAPI Bridge — Robot Data API

A ROS2-based project that publishes robot telemetry data (battery, speed, position, status)
and exposes it via a FastAPI REST API — so any web app, dashboard, or external system
can read robot data without needing ROS2.

---

## 📌 Why This Project?

In real robotics companies, robots run on ROS2 internally.
But operations teams, dashboards, and mobile apps don't speak ROS2 — they speak HTTP/REST.

This project solves that:

```
Robot (ROS2) ──► Bridge Node ──► FastAPI ──► Any Client (Web / App / Postman)
```

This is exactly how warehouse robots like QikPod expose data to operations dashboards.

## 🏗️ Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    ROS2 Layer                           │
│                                                         │
│   robot_publisher.py                                    │
│   Publishes robot data every second                     │
│          │                                              │
│          ▼                                              │
│   ROS2 Topics:                                          │
│   /robot/status    ──► "moving" / "idle" / "charging"  │
│   /robot/battery   ──► 85.5 (float)                    │
│   /robot/speed     ──► 1.2 (float)                     │
│   /robot/position  ──► "x:10.5,y:20.3" (string)        │
│          │                                              │
│          ▼                                              │
│   bridge_node.py                                        │
│   Subscribes to all topics                              │
│   Stores latest data in memory                          │
└─────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────┐
│                   FastAPI Layer                         │
│                                                         │
│   main.py                                               │
│   GET  /robot/status    ──► Returns robot status        │
│   GET  /robot/battery   ──► Returns battery level       │
│   GET  /robot/position  ──► Returns x,y coordinates     │
│   GET  /robot/all       ──► Returns everything at once  │
│   POST /robot/command   ──► Send command to robot       │
└─────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────┐
│                   Client Layer                          │
│   Postman / Web Dashboard / Mobile App / curl           │
└─────────────────────────────────────────────────────────┘
```

## ⚙️ One-Time System Setup

```bash
# Step 1 — Update system
sudo apt update && sudo apt upgrade -y

# Step 2 — Install ROS2 dev tools
sudo apt install -y \
  python3-colcon-common-extensions \
  python3-rosdep \
  python3-pip \
  python3-vcstool \
  python3-argcomplete \
  build-essential \
  cmake \
  git

# Step 3 — Initialize rosdep (run only once ever)
sudo rosdep init
rosdep update

# Step 4 — Install Python packages
pip install fastapi uvicorn ultralytics opencv-python \
  numpy requests pydantic \
  --user --break-system-packages

# Step 5 — Add ROS2 to bashrc (run only once)
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## 📦 ROS2 Package Files Explained

### `package.xml`
Defines your ROS2 package metadata.
Tells ROS2 what your package is named, who maintains it, and what it depends on.
```xml
<depend>rclpy</depend>     <!-- Python ROS2 client library -->
<depend>std_msgs</depend>  <!-- Standard ROS2 message types (String, Float32, etc.) -->
```

### `setup.py`
Tells colcon which Python scripts are ROS2 nodes.
Every node you create must be registered here to run with `ros2 run`.
```python
entry_points={
    'console_scripts': [
        'robot_publisher = robot_bridge.robot_publisher:main',
        'bridge_node = robot_bridge.bridge_node:main',
    ],
},
```

### `setup.cfg`
Basic Python package configuration.
Tells Python where to find your scripts.

### `__init__.py`
Makes the folder a Python module.
Required by Python — usually left empty.

## 🔑 Key ROS2 Concepts Used

| Concept | What It Does | Used In |
|---------|-------------|---------|
| **Node** | A single ROS2 process/program | robot_publisher, bridge_node |
| **Topic** | Named channel for data | /robot/battery, /robot/status |
| **Publisher** | Sends data to a topic | robot_publisher.py |
| **Subscriber** | Reads data from a topic | bridge_node.py |
| **Message Type** | Format of data on topic | std_msgs/String, Float32 |

## 🛠️ Daily Development Commands

```bash
# Build after any code change
colcon build --packages-select robot_bridge

# Source after every build
source install/setup.bash

# Run a node
ros2 run robot_bridge robot_publisher

# Check topics are publishing
ros2 topic list
ros2 topic echo /robot/battery

# Run FastAPI server
cd fastapi_server
uvicorn main:app --reload --port 8000
```

## 🚨 Common Mistakes to Avoid

| Mistake | Correct Way |
|---------|------------|
| Running `colcon build` inside `src/` | Always run from workspace root |
| Forgetting `source install/setup.bash` | Run after every build |
| Using venv with ROS2 | Use `pip install --user` instead |
| Naming package `package_name` literally | Use a real name like `robot_bridge` |
| Running `ros2 run` without building first | Always build then source first |
